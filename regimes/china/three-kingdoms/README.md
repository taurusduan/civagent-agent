# 三国 · 三国并立 (Three Kingdoms)

> 220-280 AD · Three Rival Kingdoms

## 简介

三国时期（220-280）是中国历史上最著名的分裂时代，也是文学作品中被演绎最多的时代。
东汉末年群雄割据，最终形成魏、蜀、吴三足鼎立的格局。

### 三国各自的特色

**魏国（曹魏）** — 务实高效
- 曹操"唯才是举"，不拘一格用人
- 推行屯田制，重视实际效益
- 继承汉制，行政体系最完善
- 占据中原，实力最强

**蜀国（蜀汉）** — 仁义忠信
- 刘备以"仁义"立国，自称汉室正统
- 诸葛亮"鞠躬尽瘁，死而后已"，是忠诚与智慧的化身
- 《出师表》《隆中对》成为千古名篇
- 偏安巴蜀，以小博大

**吴国（东吴）** — 灵活多变
- 孙权善于用人，"能用众力，则无敌于天下"
- 善于水战和外交，赤壁之战联蜀抗魏
- 开发江南，发展海上贸易
- 割据江东，灵活务实

### 竞合博弈

三国之间的关系是中国历史上最经典的多方博弈：
- **孙刘联盟**：赤壁之战联合抗曹
- **魏吴联盟**：关羽北伐时吕蒙白衣渡江
- **三方混战**：夷陵之战后三国各自为政
- 联盟关系动态变化，没有永远的敌人和朋友

## 编排模式

**Federated Autonomy (联邦自治)**: 三个独立团队各自为政，可以竞争、联盟或辩论。
类似于微服务架构中多个独立服务团队的协作模式，或者开源社区中多个fork的竞争与合并。

### 三种协作模式

1. **竞争模式** — 三国分别独立完成相同任务，择优采纳
2. **联盟模式** — 两方或三方联合攻克复杂问题，分工协作
3. **辩论模式** — 三方各提方案，互相评审，择优而行

## 适用场景

- 需要多方案比较和竞争提质的项目
- 需要不同视角（效率型/质量型/创新型）的复杂问题
- 适合A/B测试思维的产品开发
- 需要红蓝对抗或多方辩论的安全审查项目
